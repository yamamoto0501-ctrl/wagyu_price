# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
リモートリポジトリは、インターネット上の別のサーバーやクラウドサービス上に存在するプロジェクトのコピーです.
ローカルリポジトリは、自身のコンピュータ上に存在する、プロジェクトのすべてのファイルと変更履歴の完全なコピーです。


## プッシュとマージの違いは何でしょうか？
プッシュはローカルで変更したコミット履歴をリモートリポジトリに「送り出す操作」で、マージは複数のブランチの変更履歴を「統合・合流させる」操作です。


## コミットとプッシュの違い
コミットはローカル環境で加えた変更を履歴として記録する行為で、「作業の区切り」や「スナップショットの保存」です。一方、プッシュはそのローカルリポジトリに記録された変更履歴を、GitHubなどの「リモートリポジトリ」に送り出して反映させる操作です。


## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
コミットはどんな編集を行なったのかを示すメッセージなので、
編集内容をより正確に表すようなコミットメッセージにする。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
ローカルでマージしたのをmasterブランチにその変更履歴を取り込んで変更を反映させるもの。
プルリクエストとは 変更内容をマージをしてくださいという依頼を行うこと。
ローカルは作業者だけで完結できるが、プルリクエストでは完結できない。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？
コンフリクトを起こした場合は
「先にマージされた変更内容を取り込む」
「後にマージしようとしている変更内容を取り込む」
「どちらの変更内容も取り込む」
